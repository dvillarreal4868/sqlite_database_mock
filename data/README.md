# Data Directory

This directory contains all project data, organized into **raw** source files and **processed** outputs.

## Directory Structure

```
data/
├── raw/
│   ├── irb_2025_001/
│   │   ├── ct/
│   │   ├── ehr/
│   │   └── mri/
│   ├── irb_2025_002/
│   │   ├── ct/
│   │   ├── ehr/
│   │   └── mri/
│   └── irb_2025_003/
│       ├── ct/
│       ├── ehr/
│       └── mri/
├── processed/
└── README.md
```
## Raw Data

> **Do not modify files in `raw/`.** All source data should remain unchanged to ensure reproducibility.

Raw data is organized by IRB protocol. Each IRB directory contains patient-level data across three modalities:

| Subdirectory | Contents |
|---|---|
| `ct/` | CT scan imaging data |
| `ehr/` | Electronic health records |
| `mri/` | MRI scan imaging data |

### Active IRB Protocols

| Directory | Protocol | Status | Notes |
|---|---|---|---|
| `irb_2025_001/` | IRB-2025-001 | Active | — |
| `irb_2025_002/` | IRB-2025-002 | Active | — |
| `irb_2025_003/` | IRB-2025-003 | Active | — |

To add a new IRB protocol, create a directory under `raw/` following the naming convention `irb_YYYY_NNN/` and include `ct/`, `ehr/`, and `mri/` subdirectories.

## Processed Data

The `processed/` directory contains outputs generated by running project scripts against the raw data. These files can be regenerated at any time from the contents of `raw/`. Refer to the project's main documentation for the relevant processing pipelines.

## Usage Notes

- All patient data is governed by its respective IRB protocol. Handle accordingly.
- When adding new raw data, update the **Active IRB Protocols** table above.
- Do not commit large imaging files to version control. Use `.gitignore` or a data versioning tool (e.g., DVC, Git LFS) as appropriate.
